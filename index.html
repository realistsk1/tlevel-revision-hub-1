🏆 Assessment Focus Overview

💡
The mark scheme uses 4 bands (0–3) for most criteria. Band 3 = Distinction. Key words that examiners look for: comprehensive, thorough, perceptive, convincing, fully, consistently.

Task 1A Decomposing the Problem 9 "Fully identifies… effectively decomposes"
Task 1A Appreciation of Wider Issues 9 "Comprehensive lines of reasoning that fully justify"
Task 1A Appreciation of Business Context 6 "Comprehensive and perceptive definitions"
Task 1B Interface Design 6 "Sophisticated and highly effective use"
Task 1B Algorithm Design 4+6 "Comprehensive… precise logic… efficient structure"
Task 1B Data Requirements 6 "Fully appropriate… thoroughly consistent naming"
Task 1B Test Strategy 6 "Thorough and detailed understanding"

🧠 Distinction Mindset

🔍

Be Comprehensive

Don't just identify — decompose. Don't just mention risks — explain how you'll mitigate each one with specific technical solutions.

🧩

Justify Everything

Every design decision needs a reason. "I used prepared statements because they prevent SQL injection attacks by separating data from SQL commands."

📐

Show Iteration

Distinction marks require evidence of iterative development — version history, regression testing, documented rationales for changes.

⚖️

Cover All Regulations

GDPR, WCAG 2.1, OWASP, Computer Misuse Act — mention them by name and explain how your solution addresses each requirement.

🧪

Test Thoroughly

Use normal, erroneous, and boundary data. Include evidence of regression testing. Link test results back to requirements.

💬

Write Precisely

Use correct technical terminology consistently. Examiners reward "consistent appropriate use of technical language" for all communication marks.
