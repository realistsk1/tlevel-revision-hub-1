📄 HTML — Semantic Form

Semantic HTML improves accessibility and SEO. Always use appropriate input types and label associations.

HTML 
<!-- Semantic registration form with accessibility attributes -->
<form id="registerForm" novalidate aria-label="User registration form">
  <fieldset>
    <legend>Create Your Account</legend>

    <div class="form-group">
      <!-- label 'for' MUST match input 'id' for accessibility -->
      <label for="email">Email Address <span aria-label="required">*</span></label>
      <input
        type="email"
        id="email"
        name="email"
        required
        autocomplete="email"
        aria-required="true"
        aria-describedby="emailError"
      />
      <span id="emailError" role="alert" aria-live="polite"></span>
    </div>

    <div class="form-group">
      <label for="password">Password *</label>
      <input
        type="password"
        id="password"
        name="password"
        required
        minlength="8"
        autocomplete="new-password"
        aria-describedby="pwHint pwError"
      />
      <span id="pwHint">Minimum 8 characters, include a number and symbol.</span>
      <span id="pwError" role="alert"></span>
    </div>

    <button type="submit">Register</button>
  </fieldset>
</form>
💡
Exam tip: Always use aria-describedby to link error messages to inputs, aria-live="polite" for dynamic feedback, and role="alert" for errors.

⚡ JavaScript — Form Validation

JavaScript 
/**
 * Client-side form validation
 * Note: Always validate server-side too — client validation can be bypassed!
 */

const validateEmail = (email) => {
  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return regex.test(email);
};

const validatePassword = (pw) => {
  return pw.length >= 8
    && /[0-9]/.test(pw)
    && /[!@#$%^&*]/.test(pw);
};

const showError = (elementId, message) => {
  const el = document.getElementById(elementId);
  if (el) {
    el.textContent = message;
    el.style.color = '#ef4444';
  }
};

const clearError = (elementId) => {
  const el = document.getElementById(elementId);
  if (el) el.textContent = '';
};

document.getElementById('registerForm')
  ?.addEventListener('submit', (e) => {
    e.preventDefault();
    let isValid = true;

    const email = document.getElementById('email').value.trim();
    const password = document.getElementById('password').value;

    // Validate email
    if (!validateEmail(email)) {
      showError('emailError', 'Please enter a valid email address.');
      isValid = false;
    } else {
      clearError('emailError');
    }

    // Validate password
    if (!validatePassword(password)) {
      showError('pwError', 'Password must be 8+ chars with a number and symbol.');
      isValid = false;
    } else {
      clearError('pwError');
    }

    if (isValid) {
      console.log('Form valid — submit to server');
    }
});

🐘 PHP — Prepared Statements

⚠️
SQL Injection: Without prepared statements, user input can escape the SQL query and run malicious commands. Prepared statements separate SQL logic from data — the database treats user input as data only, never as executable SQL.
PHP 
<?php
/**
 * SECURE user login using PDO prepared statements
 * Prevents SQL injection attacks
 */

function loginUser(string $email, string $password, PDO $pdo): array|false
{
    // 1. Prepare: SQL template with named placeholders (never concatenate user input)
    $stmt = $pdo->prepare(
        "SELECT user_id, email, password_hash, membership_level
         FROM users
         WHERE email = :email
         LIMIT 1"
    );

    // 2. Bind parameters — PDO escapes these safely
    $stmt->bindParam(':email', $email, PDO::PARAM_STR);

    // 3. Execute
    $stmt->execute();
    $user = $stmt->fetch(PDO::FETCH_ASSOC);

    if (!$user) {
        return false; // User not found
    }

    // 4. Verify password using password_verify() — NEVER compare plain text!
    if (password_verify($password, $user['password_hash'])) {
        // 5. Regenerate session ID to prevent session fixation
        session_regenerate_id(true);
        $_SESSION['user_id']  = $user['user_id'];
        $_SESSION['email']    = $user['email'];
        return $user;
    }

    return false;
}

// Hashing when registering a new user
function hashPassword(string $plain): string
{
    // password_hash uses bcrypt by default (cost factor 10)
    return password_hash($plain, PASSWORD_BCRYPT, ['cost' => 12]);
}
?>

🗄️ SQL — DDL & DML

SQL 
-- =============================================
-- DDL (Data Definition Language) — Defines schema
-- =============================================

CREATE TABLE users (
    user_id       INT          AUTO_INCREMENT PRIMARY KEY,
    email         VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    membership    ENUM('free','premium') DEFAULT 'free',
    created_at    TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE workouts (
    workout_id  INT          AUTO_INCREMENT PRIMARY KEY,
    user_id     INT          NOT NULL,
    title       VARCHAR(100) NOT NULL,
    calories    DECIMAL(8,2),
    logged_at   TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE
);

-- Index for fast lookups
CREATE INDEX idx_workouts_user ON workouts(user_id);

-- =============================================
-- DML (Data Manipulation Language) — Queries
-- =============================================

-- INSERT new workout
INSERT INTO workouts (user_id, title, calories)
VALUES (1, 'Morning Run 5km', 320.50);

-- SELECT with JOIN to get user's workouts
SELECT
    u.email,
    w.title,
    w.calories,
    w.logged_at
FROM workouts w
INNER JOIN users u ON w.user_id = u.user_id
WHERE u.user_id = 1
ORDER BY w.logged_at DESC;

-- Aggregate: total calories this week
SELECT
    SUM(calories) AS weekly_calories,
    COUNT(*) AS workout_count
FROM workouts
WHERE
    user_id = 1
    AND logged_at >= DATE_SUB(NOW(), INTERVAL 7 DAY);

🛡️ PHP — CSRF Protection

PHP 
<?php
/**
 * CSRF (Cross-Site Request Forgery) Protection
 * Attacker tricks user's browser into making an unwanted request.
 * Solution: include a secret token in every form that only our server knows.
 */

session_start();

// Generate a CSRF token once per session
if (empty($_SESSION['csrf_token'])) {
    $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
}

// Validate incoming POST requests
function validateCsrf(): void
{
    $token = $_POST['csrf_token'] ?? '';
    if (!hash_equals($_SESSION['csrf_token'], $token)) {
        http_response_code(403);
        die('CSRF validation failed');
    }
}
?>

<!-- In your HTML form: -->
<form method="post" action="/workout/create">
  <input
    type="hidden"
    name="csrf_token"
    value="<?= htmlspecialchars($_SESSION['csrf_token']) ?>"
  />
  <!-- rest of form -->
</form>

📖 Quick Reference

What is the difference between DDL and DML? ▼
DDL (Data Definition Language) defines the database structure: CREATE TABLE, ALTER TABLE, DROP TABLE. It changes the schema.

DML (Data Manipulation Language) manipulates the data within that structure: SELECT, INSERT, UPDATE, DELETE. It reads and writes rows.
Why use two languages in your solution? ▼
The mark scheme explicitly requires at least two different languages (e.g. PHP + SQL, JavaScript + PHP, Python + SQL) to reach Band 2 or above. Each language should have a clear role: PHP/Python for server-side logic, SQL for database operations, JavaScript for client-side interactivity.
What makes code "well-organised" for Distinction? ▼
According to the mark scheme, Distinction code must be: easily maintainable by a third party through consistently appropriate naming conventions, logical organisation, and informative commenting. Avoid nested if statements with copy-pasted procedural code. Use functions, classes, and modular structure.
Explain recursive algorithms with a stopping mechanism ▼
A recursive function calls itself. The mark scheme specifically warns: "if used in PHP here must be a mechanism (IF statement, etc.) that stops the recursion". This is the base case. Without it, you get a stack overflow.

function factorial(int $n): int {
  if ($n <= 1) return 1; // BASE CASE — stops recursion
  return $n * factorial($n - 1);
}
