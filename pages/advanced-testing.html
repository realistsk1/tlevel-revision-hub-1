🧪 Advanced Testing

Root Cause Analysis & Automated Testing

The Five Whys technique for defect analysis, plus automated testing tools for UI, performance, load, and accessibility — with exam-ready justifications.

🎯
Mark scheme connection: Distinction testing evidence requires "comments show a comprehensive understanding of how errors/problems were identified and how they were rectified" and evidence of "an effective iterative development process." Root Cause Analysis directly demonstrates this — it shows you didn't just fix symptoms, but identified underlying causes.

🔍 Root Cause Analysis — The Five Whys

Developed by Toyota, the Five Whys technique iteratively asks "Why?" to dig past symptoms and find the underlying cause of a defect. You don't always need exactly five — stop when you reach an actionable root cause.

⚠️
Symptom vs Root Cause: Fixing a symptom means the same problem will recur. Fixing the root cause eliminates the class of problem permanently. A patch is fast; root cause analysis is professional.

Example: BMI Classification Bug (Defect DEF-01)

1
Problem (What happened?)
BMI = 18.5 was classified as "Underweight" instead of "Normal weight"
Discovered during boundary value testing (T-06) — test data: height 170cm, weight 53.6kg
2
Why? #1
The condition was if ($bmi < 18.5) → Underweight, which is false for exactly 18.5 but the next condition elseif ($bmi < 25) captured it as Normal — wait, that should be correct. Let me check the logic again.
Re-read: actually the conditions were < 18.5 → Underweight, <= 18.5 → Normal. Overlapping boundary.
3
Why? #2
The floating point representation of 18.5 stored as a DECIMAL(5,2) rounds in a way that $bmi === 18.5 evaluates to false in PHP due to floating point precision.
var_dump(0.1 + 0.2 == 0.3); → bool(false) — classic floating point issue
4
🎯 Root Cause
Strict equality or direct comparison of floating point numbers is unreliable in any programming language due to IEEE 754 representation. The code used direct comparison where a range check was needed.
Fix: use >= 18.5 && < 25.0 ranges rather than exact equality, and round to 1 decimal place before comparison using round($bmi, 1).
💡
How to write this in your test documentation (Distinction level): "Defect DEF-01 was identified through boundary value testing at the Normal/Underweight boundary (BMI = 18.5). Root Cause Analysis revealed the defect was caused by floating point precision issues in PHP when comparing DECIMAL database values — a systemic issue affecting all boundary classifications. Fix applied: all BMI comparisons now use round($bmi, 1) before comparison, and strict range operators (>= and <) replace equality checks. Regression testing confirmed all 6 BMI boundary tests pass after the fix with no new defects introduced."

Five Whys — Worked Example 2: Security Incident

1
Problem
A student was able to cancel another student's booking by manipulating the booking_id POST parameter
2
Why #1
The cancel query only checked the booking_id, not whether the booking belonged to the logged-in user
3
Why #2
The developer assumed the booking_id would always come from the student's own list — no adversarial thinking applied
4
🎯 Root Cause
Insufficient authorisation check — authentication (are you logged in?) was present but authorisation (are you allowed to do THIS specific action?) was missing. Fix: add AND user_id = ? to the UPDATE query binding the session user_id.

⚡ Automated Testing Tools

The mark scheme mentions "performance, stress, and load testing" can be "achieved through the use of testing software." Using real tools demonstrates professional practice and earns Distinction marks.

🔍 Accessibility Testing

Browser Extension

Axe DevTools

Chrome/Firefox extension by Deque. Audits a page against WCAG 2.1 A and AA criteria. Returns categorised violations with impact level (critical, serious, moderate, minor) and exact element location. Free, accurate, industry standard.

Browser Built-in

Lighthouse (Chrome)

F12 → Lighthouse tab. Audits Accessibility, Performance, Best Practices, SEO. Returns 0–100 score. Distinction target: Accessibility score ≥ 90. Run in incognito to avoid extension interference.

Screen Reader

NVDA (Windows)

Free screen reader for Windows. Test your app by tabbing through it with eyes closed — if NVDA announces elements meaningfully, your ARIA implementation is correct. Essential for genuine accessibility testing.

Colour

WebAIM Contrast Checker

Enter foreground/background hex codes — returns contrast ratio and AA/AAA pass/fail. Normal text must be ≥ 4.5:1. Large text (18pt+) must be ≥ 3:1. Free at webaim.org/resources/contrastchecker.

🚀 Performance & Load Testing

Performance

Google Lighthouse

Performance score 0–100. Key metrics: FCP (First Contentful Paint) — time to first content. LCP (Largest Contentful Paint) — target < 2.5s. CLS (Cumulative Layout Shift) — visual stability. TTI (Time to Interactive). Distinction: ≥ 90 score.

Load Testing

Apache JMeter

Free, open-source load testing tool. Simulates hundreds/thousands of concurrent users sending HTTP requests to your server. Reports response times, throughput, error rate. Used to test NFR-02 (e.g. "99.5% uptime under 500 concurrent users").

Load Testing (Cloud)

k6 (Grafana)

Modern load testing tool — write tests in JavaScript. More developer-friendly than JMeter. Can run locally or in the cloud. Supports scripted user journeys (login → browse → book → logout).

API Testing

Postman

Test REST API endpoints manually or automatically. Write test scripts that verify response status codes, JSON structure, and data values. Can run as a full test suite (Newman CLI) in CI/CD pipelines.

🖥️ UI & End-to-End Testing

E2E Testing

Cypress

Automated browser testing — writes tests that control a real browser. Tests user journeys end-to-end: cy.get('#email').type('user@test.com'); cy.get('button[type=submit]').click(); cy.contains('Dashboard');. Fast feedback, great for regression testing.

Unit Testing (PHP)

PHPUnit

The standard PHP unit testing framework. Write test classes that call your functions/methods with known inputs and assert expected outputs. Essential for testing calculation functions like BMI, BMR, calorie algorithms in isolation.

Unit Testing (JS)

Jest

Facebook's JavaScript testing framework. Test your JS validation functions: test('validates email format', () => { expect(validateEmail('bad')).toBe(false); expect(validateEmail('a@b.com')).toBe(true); });

Security

OWASP ZAP

Free automated security scanner — crawls your web app and tests for OWASP Top 10 vulnerabilities including SQL injection, XSS, CSRF, and broken authentication. Use passively during development to catch security issues early.

PHPUnit — Unit Test Example

PHP — PHPUnit Test for BMI Calculator 
<?php
/**
 * Unit tests for the BMI calculation function.
 * Tests: normal data, boundary values, erroneous data.
 * Maps to: FR-04, Test IDs T-06 through T-12
 */
use PHPUnit\Framework\TestCase;

class BmiCalculatorTest extends TestCase
{
    // ── Normal data tests ────────────────────────────────────
    public function testNormalWeightClassification(): void
    {
        $result = calculateBmi(175, 70);
        $this->assertEquals('Normal weight', $result['classification']);
        $this->assertEquals(22.9, $result['bmi']);
    }

    // ── Boundary value tests ─────────────────────────────────

    /** @dataProvider bmiProvider */
    public function testBmiClassifications(float $h, float $w, string $expected): void
    {
        $result = calculateBmi($h, $w);
        $this->assertEquals($expected, $result['classification'],
            "BMI {$result['bmi']} should be '$expected'");
    }

    public static function bmiProvider(): array
    {
        return [
            // [height_cm, weight_kg, expected_classification]
            'underweight boundary'     => [170, 53.2, 'Underweight'],    // BMI 18.4
            'normal lower boundary'    => [170, 53.5, 'Normal weight'],  // BMI 18.5 ← key boundary
            'normal upper boundary'    => [170, 71.9, 'Normal weight'],  // BMI 24.9
            'overweight lower boundary'=> [170, 72.3, 'Overweight'],     // BMI 25.0
            'obese lower boundary'     => [170, 86.7, 'Obese'],          // BMI 30.0
        ];
    }

    // ── Erroneous data tests ─────────────────────────────────
    public function testZeroHeightThrowsException(): void
    {
        $this->expectException(\InvalidArgumentException::class);
        calculateBmi(0, 70);
    }

    public function testNegativeWeightThrowsException(): void
    {
        $this->expectException(\InvalidArgumentException::class);
        calculateBmi(170, -5);
    }
}
// Run: vendor/bin/phpunit tests/BmiCalculatorTest.php
