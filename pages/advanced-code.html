🌐 REST API Integration & JSON Parsing

The mark scheme explicitly mentions "Web service APIs and parsing JSON/XML to service a complex client-server model" as evidence for Distinction functionality marks.

💡
What is a REST API? A REST (Representational State Transfer) API is a web service that accepts HTTP requests and returns data — usually JSON. Your app sends a request to the API endpoint; the API returns structured data your code then parses and uses. Example: sending a GET request to https://api.wger.de/api/v2/exercise/ returns a JSON list of exercises.

JavaScript — Fetch API (Client-Side)

JavaScript — Fetch + JSON Parse 
/**
 * Fetches exercises from the wger fitness API and renders them.
 * Demonstrates: async/await, fetch, JSON parsing, error handling.
 */
async function loadExercises(category = 'cardio') {
  const statusEl = document.getElementById('exercise-status');

  try {
    // Show loading state (good UX)
    statusEl.textContent = 'Loading exercises…';
    statusEl.setAttribute('aria-busy', 'true');

    // Build URL with query parameters
    const url = new URL('https://api.wger.de/api/v2/exercise/');
    url.searchParams.set('format', 'json');
    url.searchParams.set('language', '2'); // English
    url.searchParams.set('category', category);
    url.searchParams.set('limit', '10');

    // Make the HTTP GET request
    const response = await fetch(url, {
      method: 'GET',
      headers: { 'Accept': 'application/json' }
    });

    // Always check HTTP status — fetch doesn't throw on 4xx/5xx
    if (!response.ok) {
      throw new Error(`API error: ${response.status} ${response.statusText}`);
    }

    // Parse JSON response body
    const data = await response.json();

    // data.results is an array of exercise objects
    const exercises = data.results ?? [];

    if (exercises.length === 0) {
      statusEl.textContent = 'No exercises found for this category.';
      return;
    }

    // Render exercises to the DOM
    const html = exercises.map(ex => `
      <div class="exercise-card">
        <h3>${sanitise(ex.name ?? 'Unnamed')}</h3>
        <p>${sanitise(ex.description ?? 'No description')}</p>
      </div>
    `).join('');

    document.getElementById('exercise-list').innerHTML = html;
    statusEl.textContent = '';

  } catch (err) {
    // Graceful degradation — show user-friendly message, log technical detail
    statusEl.textContent = 'Could not load exercises. Please try again later.';
    console.error('Exercise API error:', err);
  } finally {
    statusEl.setAttribute('aria-busy', 'false');
  }
}

// Sanitise output to prevent XSS from API data
function sanitise(str) {
  const div = document.createElement('div');
  div.textContent = str;
  return div.innerHTML;
}
⚠️
Security note: Always sanitise data from external APIs before inserting into the DOM — even "trusted" APIs could be compromised. Using textContent (not innerHTML) or the sanitise function above prevents XSS from API data.

PHP — Server-Side API Call (cURL)

PHP — cURL API request + JSON decode 
<?php
/**
 * Calls an external REST API and returns decoded JSON data.
 * Server-side API calls prevent exposing API keys to the client.
 *
 * @param  string $endpoint  Full API URL with query parameters
 * @param  string $apiKey    Bearer token or API key (optional)
 * @return array|null        Decoded JSON as associative array, or null on failure
 */
function callApi(string $endpoint, string $apiKey = ''): ?array
{
    $ch = curl_init();

    curl_setopt_array($ch, [
        CURLOPT_URL            => $endpoint,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_TIMEOUT        => 10,           // 10-second timeout
        CURLOPT_FOLLOWLOCATION => true,
        CURLOPT_SSL_VERIFYPEER => true,         // Always verify SSL in production
        CURLOPT_HTTPHEADER     => array_filter([
            'Accept: application/json',
            $apiKey ? "Authorization: Bearer $apiKey" : null,
        ]),
    ]);

    $response = curl_exec($ch);
    $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    $error    = curl_error($ch);
    curl_close($ch);

    // Log errors server-side — never expose to client
    if ($error || $httpCode !== 200) {
        error_log("API call failed: $httpCode $error → $endpoint");
        return null;
    }

    // json_decode with true = associative array (not stdObject)
    $data = json_decode($response, true);

    // Check JSON was valid
    if (json_last_error() !== JSON_ERROR_NONE) {
        error_log('Invalid JSON from API: ' . json_last_error_msg());
        return null;
    }

    return $data;
}

// Usage example
$exercises = callApi('https://api.wger.de/api/v2/exercise/?format=json&language=2');
if ($exercises !== null) {
    foreach ($exercises['results'] as $exercise) {
        echo htmlspecialchars($exercise['name']) . '<br>';
    }
}
?>

🌿 Git Version Control Workflow

The mark scheme awards Distinction iterative development documentation for "fully effective use of versioning." Git is the industry standard — know the core workflow and commit conventions.

Git — Feature Branch Workflow 
# ── Initial setup ──────────────────────────────────────────────
git init                          # Initialise repo in project folder
git remote add origin <url>       # Link to GitHub remote

# ── Everyday workflow ───────────────────────────────────────────
# 1. Always start from an up-to-date main branch
git checkout main
git pull origin main

# 2. Create a feature branch (named for the task)
git checkout -b feature/booking-conflict-check

# 3. Make changes, then stage specific files
git add student_dashboard.php includes/auth.php

# 4. Commit with a meaningful, conventional message
#    Format: <type>(<scope>): <short description>
git commit -m "feat(booking): add interval overlap conflict detection

- Uses start_date < end AND end_date > start algorithm
- Excludes cancelled/rejected bookings from conflict check
- Returns user-friendly error on conflict
Closes #12"

# 5. Push branch to remote
git push origin feature/booking-conflict-check

# 6. Create Pull Request on GitHub for code review
# 7. After review and approval, merge to main
git checkout main
git merge feature/booking-conflict-check --no-ff  # --no-ff preserves branch history
git push origin main

# 8. Tag releases for version tracking
git tag -a v1.2.0 -m "Version 1.2.0 - Conflict detection + admin filter"
git push origin v1.2.0

# ── Useful commands ─────────────────────────────────────────────
git log --oneline --graph          # Visual branch history
git diff HEAD~1                    # What changed in last commit
git stash                          # Temporarily save uncommitted changes
git revert HEAD                    # Safely undo last commit (keeps history)
git blame student_dashboard.php    # Who changed each line and when
💡
Conventional Commits for Distinction documentation: Use prefixes: feat: new feature · fix: bug fix · docs: documentation · refactor: code change without new feature · test: adding tests · security: security fix. This makes your commit history scannable evidence of your iterative process.

🏗️ Object-Oriented Programming (OOP)

The mark scheme references "use of function, procedures and Classes" for Distinction functionality. OOP organises code into objects that combine data (properties) and behaviour (methods).

📦

Encapsulation

Bundle data and the methods that operate on it together. Control access via public, protected, private — hide internal implementation details.

🧬

Inheritance

A child class inherits properties and methods from a parent class. Promotes code reuse — a PremiumUser class extends User, adding premium-only methods.

🎭

Polymorphism

Objects of different classes can be treated as the same type through interfaces or inheritance. A sendNotification() method works whether sending Email or SMS.

🔒

Abstraction

Hide complex implementation behind a simple interface. Users of a Database class call query() without knowing whether it uses MySQLi or PDO underneath.

🔌

Interfaces

Define a contract — a set of methods a class must implement. Ensures different classes can be used interchangeably if they implement the same interface.

🏭

Dependency Injection

Pass dependencies (like a DB connection) into a class rather than creating them inside. Makes classes easier to test and less tightly coupled.

PHP — OOP Example: User & Booking Classes 
<?php
/**
 * Represents a user of the Resource Hub system.
 * Demonstrates: encapsulation, constructor, typed properties, methods.
 */
class User
{
    // Private properties — encapsulation: only accessible via methods
    private int    $id;
    private string $username;
    private string $passwordHash;
    private string $role;

    /**
     * Constructor — dependency injection (PDO passed in, not created here).
     */
    public function __construct(
        int    $id,
        string $username,
        string $passwordHash,
        string $role = 'student'
    ) {
        $this->id           = $id;
        $this->username     = $username;
        $this->passwordHash = $passwordHash;
        $this->role         = $role;
    }

    // Getters — controlled read access (no setter for passwordHash — immutable)
    public function getId(): int    { return $this->id; }
    public function getUsername(): string { return $this->username; }
    public function isAdmin(): bool { return $this->role === 'admin'; }

    public function verifyPassword(string $plain): bool
    {
        return password_verify($plain, $this->passwordHash);
    }
}

/**
 * PremiumUser extends User — inheritance + polymorphism.
 * Inherits all User methods and adds premium-specific behaviour.
 */
class PremiumUser extends User
{
    private ?\DateTime $subscriptionExpiry;

    public function __construct(int $id, string $username, string $hash, ?\DateTime $expiry)
    {
        parent::__construct($id, $username, $hash, 'premium');
        $this->subscriptionExpiry = $expiry;
    }

    public function hasActiveSubscription(): bool
    {
        if ($this->subscriptionExpiry === null) return false;
        return $this->subscriptionExpiry > new \DateTime();
    }

    // Override parent method — polymorphism
    public function getPersonalisedPlan(): array
    {
        if (!$this->hasActiveSubscription()) {
            throw new \RuntimeException('Premium subscription required');
        }
        // AI-generated plan logic here...
        return [];
    }
}
?>
What is the difference between a class and an object?▼
A class is the blueprint/template — it defines properties and methods. An object is an instance of that class — a specific entity created from the blueprint. E.g. User is the class; $john = new User(1, 'jsmith', $hash) creates a specific user object. You can create many objects from one class.
Why use OOP instead of procedural code?▼
OOP makes large codebases more maintainable (encapsulation hides complexity), more reusable (inheritance prevents duplication), and more testable (objects can be isolated and tested independently). The mark scheme awards Distinction for code "easily maintainable by a third party" — OOP directly supports this.
What is a static method?▼
A static method belongs to the class itself, not to instances. Called as User::create($data) rather than $user->create($data). Useful for factory methods (creating instances), singleton patterns (ensuring only one DB connection), and utility functions that don't need instance data.
